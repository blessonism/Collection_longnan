# 周小结流程自动化优化方案

## 一、现状与问题定位

### 1.1 当前流程
```
周五群内发通知 → 领导填写 → 收集 → 人工校对 → 重命名归档 → 打印
```

### 1.2 核心痛点

| 痛点 | 具体表现 | 根因 |
|------|----------|------|
| 校对耗时 | 逐份检查格式、标点、错字 | 领导年龄跨度大，填写质量参差不齐 |
| 收集分散 | 文件散落在群聊中，需逐个下载 | 缺乏统一提交入口 |
| 重命名繁琐 | 手动按序重命名 | 无自动化工具 |
| 催交成本 | 需多次提醒未交人员 | 无提交状态追踪 |

### 1.3 可消除的摩擦点

- ✅ 手动发通知 → 定时自动发送
- ✅ 群内收集文件 → 统一上传入口
- ✅ 人工校对格式/错字 → 脚本自动检测
- ✅ 手动重命名 → 批量自动重命名
- ✅ 人工追踪提交状态 → 自动统计谁交谁没交

## 二、自动化方案设计

### 2.1 方案概览

```
┌─────────────────────────────────────────────────────────────┐
│                      周小结自动化流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [定时通知]  →  [统一收集]  →  [自动校对]  →  [批量归档]     │
│      │              │              │              │         │
│   企微/钉钉      共享文件夹      Python脚本     脚本重命名    │
│   定时消息       或简易网页       + AI校对       + 汇总       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 模块拆解

#### 模块1：通知自动化
| 方案 | 实现方式 | 难度 |
|------|----------|------|
| 企微/钉钉定时消息 | 内置功能，设置周五定时 | ⭐ |
| 机器人推送 | 企微/钉钉机器人 Webhook | ⭐⭐ |

#### 模块2：统一收集入口
| 方案 | 优点 | 缺点 | 难度 |
|------|------|------|------|
| 共享文件夹（腾讯/金山） | 零开发，领导易上手 | 无法自动追踪状态 | ⭐ |
| 在线表单+附件 | 可追踪提交状态 | 需领导适应新方式 | ⭐ |
| 简易上传网页 | 完全可控，可集成校对 | 需开发部署 | ⭐⭐⭐ |

**推荐**：先用共享文件夹，后期可升级为简易网页

#### 模块3：自动校对（核心）
| 检查项 | 技术方案 | 说明 |
|--------|----------|------|
| 格式检查 | Python-docx 读取样式 | 检查字体、字号、行距是否符合模板 |
| 标点检查 | 正则表达式 | 检测中英文标点混用、连续标点等 |
| 错字检查 | pycorrector / 百度API | 自动检测并标记疑似错字 |
| AI 润色建议 | 大模型 API（可选） | 给出修改建议，人工确认 |

**输出**：生成校对报告，标记问题位置，人工只需复核

#### 模块4：批量归档
| 功能 | 实现方式 |
|------|----------|
| 自动重命名 | Python 脚本按规则重命名（如：序号_部门_姓名_日期.docx） |
| 自动归档 | 移动到指定文件夹，按周/月分类 |
| 生成目录 | 输出文件清单，方便打印时核对 |

## 四、技术选型建议

### 4.1 脚本开发
```
语言：Python
核心库：
  - python-docx：读写 Word 文档
  - pycorrector：中文错字检测
  - re：正则表达式处理标点
  - os/shutil：文件操作
```

### 4.2 简易平台（可选）
```
框架：Streamlit（最快）或 Flask
功能：
  - 文件上传
  - 自动校对并显示结果
  - 一键下载归档文件
```

### 4.3 部署方式
- 本地运行：适合个人使用，无需服务器
- 内网部署：如需多人访问，可部署在内网服务器
