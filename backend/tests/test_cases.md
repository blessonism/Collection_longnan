# 规则检查器测试用例

复制以下内容到前端表单进行测试，验证各项规则是否正常工作。

---

## 测试用例 1：序号格式错误

**本周工作：**
```
1、完成项目报告撰写工作。
2。参加部门周例会。
3.  跟进重点任务进展。
4.4.协调资源配置。  
```

**预期检测：**
- `1、` → `1.`
- `2。` → `2.`
- `3.  ` → `3.`（多余空格）
- `4.4.` → `4.`（重复序号）

---

## 测试用例 2：英文标点错误

**本周工作：**
```
1.完成项目报告,已提交审核。
2.参加会议;讨论工作安排。
3.工作内容:完成年度总结。
4.这个问题怎么解决?
5.完成了重要任务!
```

**预期检测：**
- `,` → `，`
- `;` → `；`
- `:` → `：`
- `?` → `？`
- `!` → `！`

---

## 测试用例 3：时间格式冒号（不应报错）

**本周工作：**
```
1.上午10:30参加项目评审会议。
2.下午14:00与客户进行电话沟通。
```

**预期：** 时间格式的冒号不应被检测为错误

---

## 测试用例 4：英文括号错误

**本周工作：**
```
1.完成项目(重点任务)报告撰写。
2.参加培训(共计3天)学习新技能。
```

**预期检测：**
- `(` → `（`
- `)` → `）`

---

## 测试用例 5：斜杠错误

**本周工作：**
```
1.完成工作/会议安排。
2.协调资源/人员配置。
```

**预期检测：**
- `/` → `；`（中文语境）

---

## 测试用例 6：连续重复标点

**本周工作：**
```
1.完成项目报告。。
2.参加会议，，讨论安排。
3.跟进任务。.
```

**预期检测：**
- `。。` → `。`
- `，，` → `，`
- `。.` → `。`

---

## 测试用例 7：句末标点错误

**本周工作：**
```
1.完成项目报告；
2.参加部门会议.
3.完成了重要任务！
4.跟进重点工作
```

**预期检测：**
- `；` → `。`（句末分号应改句号）
- `.` → `。`（英文句号应改中文句号）
- `！` → `。`（公文不用感叹号）
- 第4条缺少句末标点，建议添加 `。`

---

## 测试用例 8：多余空格

**本周工作：**
```
1.完成 项目 报告撰写工作。
2.参加会议 ，讨论工作安排。
```

**预期检测：**
- `完成 项目` → `完成项目`
- `项目 报告` → `项目报告`
- `会议 ，` → `会议，`

---

## 测试用例 9：缺少序号

**本周工作：**
```
完成项目报告撰写工作。
参加部门周例会。
```

**预期检测：**
- 第1行缺少序号，建议 `1.完成项目报告撰写工作。`
- 第2行缺少序号，建议 `1.参加部门周例会。`

---

## 测试用例 10：综合测试（真实场景）

**本周工作：**
```
1、完成项目报告,已提交审核;
2.参加部门会议 ,讨论工作安排。
3.跟进重点任务/协调资源。
4.完成了年度总结撰写！
5.上午10:30参加评审会议。
```

**预期检测：**
- 第1条：`1、` → `1.`，`,` → `，`，`;` → `；`，`；` → `。`（句末）
- 第2条：` ,` → `，`（空格+逗号）
- 第3条：`/` → `；`
- 第4条：`！` → `。`
- 第5条：无错误（时间格式冒号正确）

---

## 测试用例 11：下周计划部分

**本周工作：**
```
1.完成项目报告。
```

**下周计划：**
```
1、继续推进项目,完成验收!
2.跟进重点任务；
```

**预期检测：**
- 下周计划第1条：`1、` → `1.`，`,` → `，`，`!` → `！`，`！` → `。`
- 下周计划第2条：`；` → `。`

---

## 测试用例 12：正确格式（不应报错）

**本周工作：**
```
1.完成项目报告撰写工作，已提交审核。
2.参加部门周例会，讨论下周工作安排。
3.跟进重点任务进展，协调资源配置。
4.完成年度工作总结初稿撰写。
```

**预期：** 无错误
